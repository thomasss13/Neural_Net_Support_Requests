Данный проект реализован в качестве курсовой работы, но его можно запустить со своего ПК и поиграться с параметрами. Считаю, это будет полезно, как минимум в ознакомительных целях.

Проект требует следующий стек:
1. Python 3.11-3.12 (TensorFlow пока не поддерживается на более новых версиях)
2. TensorFlow>=2.13.0
3. Scikit-learn>=1.3.0
4. Matplotlib>=3.7.0
5. Flask>=3.0.0
6. Numpy>=1.24.0

Порядок запуска (запустить через терминал IDE):
1. train.py (обучение модели)
2. app.py (веб-сервер)
3. visualize.py (графики)

Если хотите запустить модель с уже имеющимися параметрами, то через терминал запустите только app.py.

model.py запустить, конечно, тоже можно, но смысла сильно не имеет, так как веса будут переинициализированы в train.py.

Можно подергать параметры:
1. model.py
   Переменные:
   VOCAB_SIZE     - размерь словаря
   MAX_LEN        - длина вектора tokenID (максимальная длина обращения в словах)
   EMBEDDING_DIM  - длина вектора эмбеддинга
   
   Методы, экземпляры и др:
   layers.SpatialDropout1D(0.4, name="spatial_dropout") - 0.4 отвечает за то, сколько векторов эмбеддинга будет обнулено

   layers.Dense(
            128, activation="relu",                     - количество нейронов в скрытом слое (128) и функция активации (relu)
            kernel_regularizer=regularizers.L2(1e-4),   - тип регуляризации (L2) и его коэффициент (0.0001)
            name="hidden",)

   layers.Dropout(0.5, name="dropout")                  - 0.4 отвечает за то, сколько нейронов будет "выключено" (обнулено)

   model.compile(
        optimizer=keras.optimizers.Adam(learning_rate=0.001), - оптимизатор (Adam) и его learning_rate по умолчанию
        loss="sparse_categorical_crossentropy",               - функция потерь
        metrics=["accuracy"],)
   
3. train.py
   Переменные:
   EPOCHS                - количество эпох обучения по умолчанию (может остановиться раньше, так как работает EarlyStopping) 
   BATCH_SIZE            - размер батча (на сколько кусков будет разбита обучающая часть датасета для прогона в одну эпоху)
   TEST_SIZE             - процент датасета, выделяемый под тестирование модели
   RANDOM_SEED           - просто семя случайности, которое можно менять как угодно (только int), у меня это отсылка на "Автостопом по галактике"
   UNCERTAINTY_THRESHOLD - порог неуверенности, если модель получает вероятность по классу меньше данного числа, то переводит обращение оператору
   
   Методы, экземпляры и др:
   tokenizer = Tokenizer(
    num_words=VOCAB_SIZE,
    oov_token="<OOV>",                               - метка для слов, не вошедших в словарь (по умолчанию)
    lower=True,
    filters='!"#$%&()*+,-./:;<=>?@[\\]^_`{|}~\t\n',) - фильтр, который удаляет конкрутные символы из введенного текста

   early_stopping = keras.callbacks.EarlyStopping(
    monitor="val_accuracy",
    patience=15, - терпение нейросети перед остановкой обучения (сколько эпох терпим)
    restore_best_weights=True,
    mode="max",
    verbose=1,)

   reduce_lr = keras.callbacks.ReduceLROnPlateau(
    monitor="val_accuracy", - следим за точностью нейросети (можно поменять на val_loss)
    factor=0.5,             - во сколько раз увеличивается learning_rate при выходе обучения на плато
    patience=7,             - сколько ждем эпох без улучшения
    min_lr=1e-5,            - минимальный learning_rate, ниже которого алгоритм пойти не может
    mode="max",
    verbose=1,)

Также я уже успела доработать некоторые нюансы: чуть увеличила датасет и исправила заглушки в вебе (предложения обращений), также перестроила графики, поэтому они будут отличаться от тех, которые находятся в презентации. Старые данные сохранены, но переименованы (появилась приписка _old у файлов dataset.py, index_old.html и папки plots). Чтобы запустить код со старыми данными, чтобы посмотреть, как все работало до изменений, можно либо изменить названия самих файлов, либо в самом коде переписать вызываемые через ту же приписку _old.

Если кому-то будет интересно поиграть и запустить проект, то сразу стоит предупредить, что итоговые значения и графики не будут такими же, как получились у меня. Тому есть 2 причины: во-первых, эмбеддинги и веса инициализируются случайным образом, что влияет на ход обучения и вследствие того влияет на весь процесс обучения и графики, во-вторых, Dropout имеет стохастическую природу (обнужение происходит случайно по прописанной вероятности).
